Compilation et pré-processeur¶

La compilation en C se dcompose en deux tapes successives :

  1. L'excution du pr-processeur,
  2. L'excution du compilateur C.

Les deux tapes sont ralises par un seul appel la suite de compilation avec la commande :

gcc -Wall premier.c -o premier

Les options permettent :

  • Wall : d'afficher l'ensemble des avertissements produits par la compilation
  • -o : de choisir le nom de l'excutable gnr.

Note 1 : dans le cas particulier o on utiliser la bibliothque math.h, il faut rajouter -lm la commande de compilation.

Le pr-processeur fournit un unique fichier au compilateur, qui le transforme en un fichier binaire excutable. Ce pr-processeur :

  • Supprime les commentaires de ligne // ou de bloc /* */.
  • Interprte les commandes pr-processeur qui commencent par # (#define, #include, etc.)

Expressions et compatibilité entre types¶

Définition d'une expression¶

Une expression est une instruction qui est caractrise par une valeur de retour. Voici quelques exemples :

  • Une variable initialise 
    int val = 20;
   val; // La variable `val` vaut 20 dans cette instruction.
  • Une comparaison : (b > 20). Cette expression vaudra true ou false.
  • L'utilisation d'oprateur arithmtiques : 
    int x = 3;
  x + 3; // Cette expression vaut 6
  (x * 2) / 3; // Cette expression vaut 2

Note : Une affectation est aussi une expression : val = 40 est une expression qui vaut 40. L'utilisateur de l'affectation comme expression est viter en C.

Priorité des opérateurs¶

En C, la priorit des oprateurs valus dans une mme expression est la suivante :

Priorit Oprateurs
1 +, -, ! (unaires)
2 *, /(entier), /(flottant), %, &&
3 +, -, ||
4 <, >, <=, >=, ==, !=

La priorit 1 est la plus forte. Les oprateurs boolens sont prsents en gras.

Compatibilité entre types¶

En C, une expression peut tre compose d'expressions de types diffrents si ces types sont compatibles. Voici quelques exemples (fichier compatibilite.c:

In [ ]:
#include <assert.h>
#include <stdlib.h>
int main(){
    int quantite = 5;
    float prix = 12.3;

    float total = prix * quantite; //l'entier quantit est compatible avec les flottants    
    float recette = 12; // l'entier 12 est compatible avec le flottant recette.
    assert(total == 12.3*5.0 && recette == 12.0);

    quantite = total / recette;
    assert(quantite != total / recette);
    // le rel obtenue par la division de total et recette 
    // n'est pas compatible avec l'entier quantite.
    
    return EXIT_SUCCESS;
}

Si un type A est compatible avec un type B, on peut interchanger une expression de type B par une expression de type A sans changer la valeur de l'expression.

Typiquement, un type A est compatible avec un type B si le passage de l'un l'autre n'engendre pas de perte de donne :

  • Un entier est compatible avec un rel (12 devient 12.0)
  • Un rel n'est pas compatible avec un entier (le passage de 1.3 1 introduit une perte d'information).

Les types énumérés¶

Un type numr permet de dfinir un ensemble discret de valeurs possibles. L'exemple suivant dclare un type numr enum Jour :

Une variable de type enum Jour ne peut prendre que ces valeurs.

Il est conseill de crer un alias au type enum Jour l'aide de l'instruction typedef

Adresse mémoire¶

En C, il est possible de connaitre l'adresse mmoire laquelle est stocke une variable var1 avec l'oprateur unaire & (perluette) :

Déclaration d'un pointeur¶

Pour dclarer un pointeur en C, il faut connaitre le type de la donne qui sera enregistre cette adresse. Par exemple, on peut dclarer un pointeur sur une variable de type entier ou un pointeur sur une variable de type double. Pour dclarer un pointeur, on met * devant le nom de la variable :

Initialisation d'un pointeur¶

Comme pour n'importe quelle dclaration de variable en C, le pointeur n'est pas initialis une valeur par dfaut sa dclaration. En d'autres termes, l'adresse enregistre n'a aucun sens, elle est alatoire.

On doit initialiser un pointeur soit avec :

  • Soit avec l'adresse mmoire d'une variable du bon type:
    int var1 = 10;
  int* ptr_int = &var1; // initialisation avec l'adresse de la variable var1
  • Ou avec la valeur d'un pointeur de mme type
    int* ptr_int_2 = ptr_int;

Si l'adresse est n'est pas connue au moment de la dclaration, il faut toujours initialiser le pointeur NULL

Pour accder la variable pointe, on utilise aussi l'oprateur * plac avant l'identificateur.

  1. Avec l'oprateur ->
    point pt1;
 struct point * ptr_point = &pt1; 
 ptr_point->x = 12; 
 ptr_point->y = 0;

    Rgle : Il faut utiliser la notation ->

    // dclaration d'un pointeur sur la 4e case du tableau

L'oprateur - permet de se dplacer vers la gauche dans le tableau :

ptr = ptr-2;
    //ptr pointe sur la 2e case du tableau
    assert(*ptr == 40);

Les sous-programmes¶

Le langage C ne permet pas de diffrencier les fonctions des procdures algorithmiques. Le seul sous-programme utilisable est la fonction :

type_retour identificateur_fonction ( type_param1 id_param1, type_param2 id_param2, ...)

Chaque paramtre formel est typ et pass par valeur (mode IN algorithmique). Ainsi, l'appel une fonction sur des paramtres rels variables ne modifie pas la donne. Par contre, les instructions de la fonction connaissent la donne (valeur) et peuvent la manipuler pour fournir l'unique rsulat retourn via le type retour.

On passe l'adresse d'une variable une fonction l'aide d'un pointeur. Pour se faire, il faut dclarer la fonction avec des paramtres formels qui sont des pointeurs.
